## Next Steps

1. Verify the operator is running:
   $ kubectl get pods -n {{ .Release.Namespace }} -l app.kubernetes.io/name=firedoor

2. Check the operator logs:
   $ kubectl logs -n {{ .Release.Namespace }} -l app.kubernetes.io/name=firedoor

3. Create your first Breakglass resource:
   $ kubectl apply -f - <<EOF
   apiVersion: access.cloudnimbus.io/v1alpha1
   kind: Breakglass
   metadata:
     name: example-breakglass
     namespace: default
   spec:
     subjects:
     - kind: User
       name: user@example.com
     clusterRoles:
     - admin
     approvalRequired: false
     duration: 1h
     justification: "Example breakglass for testing"
   EOF

4. Monitor the breakglass status:
   $ kubectl get breakglass -A
   $ kubectl describe breakglass example-breakglass

## Troubleshooting

If you encounter issues:

1. Check the operator logs:
   $ kubectl logs -n {{ .Release.Namespace }} -l app.kubernetes.io/name=firedoor

2. Verify RBAC permissions:
   $ kubectl auth can-i create breakglass --namespace default

3. Check CRD installation:
   $ kubectl get crd breakglasses.access.cloudnimbus.io

4. Review the operator status:
   $ kubectl describe deployment firedoor-controller-manager -n {{ .Release.Namespace }}
